*render-markdown-mermaid.txt*  Inline mermaid rendering for render-markdown.nvim

render-markdown-mermaid.nvim renders fenced |mermaid| code blocks inline with
|render-markdown.nvim| by calling Beautiful Mermaid (`bm`) or
`mermaid-ascii`.
The default setup attaches to both Markdown and MDX buffers.

INSTALL

Use your plugin manager to install:

>
    {
        'cavanaug/render-markdown-mermaid.nvim',
        dependencies = {
            'nvim-treesitter/nvim-treesitter',
            'MeanderingProgrammer/render-markdown.nvim',
        },
        build = ':TSUpdate markdown markdown_inline',
        opts = {},
    }
<

This uses the plugin defaults and automatically configures
|render-markdown.nvim|.

HEALTH

Use:

>
    :checkhealth render-markdown-mermaid
<

This verifies that `render-markdown.nvim`, `bm`, and `mermaid-ascii` are
available, then reports which renderer will be selected by default.
It also checks for the `markdown` and `markdown_inline` treesitter parsers.

Note that |render-markdown.nvim| may still warn about optional LaTeX support if
the `latex` parser or a converter such as `utftex` / `latex2text` is not
installed. Those warnings only matter if you want LaTeX rendering.

OPTIONS

The plugin exposes `require('render-markdown-mermaid').setup(opts)` and, when
using lazy.nvim, `opts = {}` maps directly to that setup table.

Default values: >
    {
        mode = 'unicode',
        placement = 'above',
        replace = false,
        cmd = { 'bm' },
        auto_setup_render_markdown = true,
        debounce = 150,
        timeout = 2000,
        cache = true,
        hide_source = false,
        max_block_lines = 200,
        render_markdown = {
            file_types = { 'markdown', 'mdx', 'markdown.mdx' },
        },
        cli = {
            border_padding = 1,
            padding_x = 5,
            padding_y = 5,
        },
    }
<

The default renderer selection prefers `bm` and falls back to
`mermaid-ascii`. Set `cmd = { ... }` to override that choice explicitly.

`render_markdown` is an optional passthrough table forwarded to
|render-markdown.nvim| when this plugin auto-configures it.

`placement` controls whether the rendered diagram is shown `above` or `below`
the raw mermaid fence while editing. The default is `above`.

If `replace = true`, the raw mermaid fence is visually replaced in normal mode
whenever the cursor is outside that block. Entering that block or entering
insert mode reveals the source again and falls back to `placement`.

`mode` controls whether text renderers use Unicode box-drawing characters
(`unicode`) or plain ASCII fallback characters (`ascii`). The default is
`unicode`.

If `hide_source = true`, the raw mermaid fence is concealed whenever the cursor
is outside the fence, while the rendered diagram stays visible.
